037_how to configure VRF in CloudIL

Step-by-step guide

1.    Create the routing instance

To create a routing instance, enter the following command:

add networking vrf name <name> [id <id>]

NOTE: Vrf id 0~9 is reserved by system, operator shall use vrf id range 10~63.

2.    Verify the routing instance

To check the existing routing instances, enter the following command:

show networking vrf

3.    Move interface to a routing instance

Move the interface to a instance by using the following command:

set networking ether <owner> newvrf <newvrfid>

NOTE: move one interface to new vrf should delete the ip address on the interface first, except the IP address with set by system: yes attribute. The IP address with this attribute will also move to new vrf automatically. If the new VRF also exist the same IP address, the moving to new VRF operation will failed.

4.     Linux CLI to show interface in specific namespace(VRF)

# ip netns exec vrf10 ip addr

 

If your interface and IP address was shown here, the VRF is working….   I have tried to configure VRF successfully like this! J  Good Luck.
