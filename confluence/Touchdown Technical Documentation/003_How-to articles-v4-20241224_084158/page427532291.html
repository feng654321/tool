025_How to patch for IL Iu/lub signaling PRBs and RPM

  • Patch commands
  • How to patch IL Iu signaling PRBs in mcRNC

  • How to patch IL Iu signaling PRBs in ADA
  • How to patch with RPM
  • How to remove patch

 

Patch commands

  1. Install patch
    fsswcli -p  -i | --install [<D>/<A>/<V>:]{[<source>=<target>]

    • <D> Delivery, for example: R_IL_14.6.1.36.bcn.r.20150121.47

    • <A> Architecture, for example: x86_64/mips

    • <V> Variant, for example: std/wnd 

      Note: <D>/<A>/<V> can be replace by @ and *  

            @: Current

            *: All

  2. Remove patch
    fsswcli -p  –r <patch name>
  3. List patch
    fsswcli -p -l

How to patch IL Iu signaling PRBs in mcRNC

  1. Copy patch files to mcRNC
    for example:
    • cp sagpro and sorpro to  /root/lfang/ of mcRNC
  2.  Add execution right: 
    • chmod +x /root/lfang/sagpro
    • chmod +x /root/lfang/sorpro
  3. Check where is PRB located by SCLI
    for example
    • fshascli -v | grep sagpro, it will show
      command=(/opt/nokiasiemens/SS_ILSigss7/bin/sagpro)
    • fshascli -v | grep sorpro, it will show
      command=(/opt/nokiasiemens/SS_ILSigss7/bin/sorpro)

  4. Lock the relative RGs of patch files, if you cannot find the relative RG in the below table, you can send mail to I_EXT_MBB_WCDMA_SWD3_SECTION_FOX_GMS DG

    IU PRBs and their relative RG

    PRBsRelative RGtype
    sagproQNIUwnd
    sorpro, segpro, iuvprb, iurprb, iapprbQNUSCPstd
    scxadm, smnpro, sptprb, rc3prbQNCFCPwnd
    so7prbQNOMUwnd

     

    IUB PRB and their relative RG

    PRBsRelative RGtype
    nbmpro QNCFCPwnd
    nblstc QNIUBwnd
    hlpnbl QNEITPwnd
    nbapcQNCSCPstd

     

     

     

  5. Try to patch files with following commands

    • fsswcli -p -i @/@/wnd:/root/lfang/sagpro=/opt/nokiasiemens/SS_ILSigss7/bin/sagpro

      • mips64 wnd (corresponding db build for CFPU/CLA EIPU nodes)

    • fsswcli -p -i @/@/std:/root/lfang/sorpro=/opt/nokiasiemens/SS_ILSigss7/bin/sorpro

      • mips64 std(corresponding db build for USPU)

  6. Check the patch validity by md5sum
    for example:
    • md5sum /opt/nokiasiemens/SS_ILSigss7/bin/sagpro
    • md5sum /root/lfang/sagpro
    • ssh uspu-0
    • md5sum /opt/nokiasiemens/SS_ILSigss7/bin/sorpro
  7. Unlock the RGs of patch files
  8. Check if all RUs startup by command
    • ilclifunit -u

     In case the RGs cannot be locked, please try the following steps:

  1. Copy patch files to mcRNC
    for example:
    • cp scxadm to  /root/lfang/ for mips64 wnd (corresponding db build for CFPU/CLA EIPU nodes)
  2.  Add execution right: 
    • chmod +x /root/lfang/scxadm
  3. Check where is PRB located by SCLI
    for example
    • fshascli -v | grep scxadm, it will show
      command=(/opt/nokiasiemens/SS_ILSigss7/bin/scxadm)

  4. Try to patch files with following commands

    • fsswcli -p -i --skip-mount @/@/wnd:/root/lfang/scxadm=/opt/nokiasiemens/SS_ILSigss7/bin/scxadm

  5. Restart the system

    • fsclish -c "set has restart force managed-object / "

  6. Check the patch validity by md5sum
    • md5sum /opt/nokiasiemens/SS_ILSigss7/bin/scxadm
    • md5sum /root/lfang/scxadm

  7. Check if all RUs startup by command 

    • ilclifunit -u

How to patch with RPM

  1. Copy patch files to mcRNC/MGW/ADA
    for example:
    • cp *.rpm to /root/lfang/ for mcRNC
  2. Try to patch rpm with following commands
    • fsswcli -p -i --skip-mount @/@/*:/root/lfang/***.rpm

  3. Restart the system

    • fsclish -c "set has restart force managed-object / "

  4. Check if all RUs startup by command 

    • ilclifunit -u

How to remove patch

  1. Lock all RGs
  2. Use fsswcli -p --list and fsswcli -p -r to remove patch
    for example
    • Find patch name by fsswcli -p -l | grep -i tag

    • remove prb patch by fsswcli -p -r with prb patch name

    • remove rpm patch by fsswcli -p -r with rpm patch name
      fsswcli -p -r --skip-mount tag: patch name
  3. Unlock RGs. Then the system will be back to before as you did not patch TI of PRBs.

In case the RGs cannot be locked or remove rpm patch, please try the following steps:

  1. use fsswcli -p -r --skip-mount to remove patch
  2. Restart the system

    • fsclish -c "set has restart force managed-object / "

  3. Wait system up and check if all RUs startup

    • ilclifunit -u
