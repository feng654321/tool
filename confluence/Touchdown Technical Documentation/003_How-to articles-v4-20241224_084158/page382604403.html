013_How to configure traffic env on CloudIL

● 底层traffic配置

1 vFPC

1 确定traffic接口状态为 up

#ip addr 

2 在traffic接口上绑定ip地址

#ip addr add 29.1.14.211/24 dev traffic
#ip addr add 29.1.14.210/24 dev traffic

 

其他相关命令

※ 确定接口状态
#ip addr -- 输出详细的接口信息

 

7: traffic: <BROADCAST,MULTICAST,UP,LOWER_UP（接口状态）> mtu 1500 qdisc pfifo_fast state UP qlen 1000
    link/ether 52:54:00:c5:e6:6e brd ff:ff:ff:ff:ff:ff
    inet 29.1.14.101/24 scope global traffic
    inet 29.255.255.1/24 scope global traffic
    inet 29.1.6.101/24 scope global traffic
    inet 29.1.14.102/24 scope global secondary traffic
    inet 29.1.14.211/24 scope global secondary traffic
    inet 29.1.14.210/24 scope global secondary traffic
    inet 29.1.6.102/24 scope global secondary traffic
    inet6 fe80::5054:ff:fec5:e66e/64 scope link

 

#ip link -- 输出接口状态
7: traffic: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP qlen 1000
※ 接口状态设置
# ip link set traffic up
# ip link set traffic down
※接口上删除IP地址
#ip addr delete 29.1.14.211/24 dev traffic
#ip addr delete 29.1.14.210/24 dev traffic
※ 接口绑定IP地址
#ip addr add 29.1.14.211/24 dev traffic
#ip addr add 29.1.14.210/24 dev traffic
2 vRNC

1 确定node eipu上的traffic接口状态为up

#fsclish -c "show networking interface iface up"

接口名称有可能是traffic、up等等，不同版本会有不同的名称，请注意确认

2 在traffic接口上绑定ip地址

#fsclish -c "add networking address /QNUP-0 ip-address 29.1.14.200/24 iface up"
#fsclish -c "add networking address /QNUP-0 ip-address 29.1.14.201/24 iface up"

 

其他相关指令

※ 确认接口状态
#fsclish -c "show networking interface iface up" 
--------------------------------------------------------------
interfaces in default instance:
-------------------------------
  up (Ethernet interface)
    VRF name(ID)        : default (0)

#fsclish -c "set networking interface /EIPU-0 iface up admin down"
#fsclish -c "set networking interface /EIPU-0 iface up admin up"

※ 接口绑定的IP地址信息
# fsclish -c "show networking address iface up"
※ 接口IP地址的增删

◎ 删除
注意先需要删除IP与ipbr的映射IPRO
#fsclish -c "delete networking ipro ip-address 29.1.14.200 ipbr-id 2 iface up owner /QNUP-0"
#fsclish -c "delete networking ipro ip-address 29.1.14.201 ipbr-id 1 iface up owner /QNUP-0"

◎ 增加
#fsclish -c "add networking address /QNUP-0 ip-address 29.1.14.200/24 iface up"
#fsclish -c "add networking address /QNUP-0 ip-address 29.1.14.201/24 iface up"
3 ip route check
1 vRNC --> vFPC
在vRNC上需要登录到EIPU-0
ping 29.1.14.211
ping 29.1.14.210
2 vRNC <-- vFPC
ping  29.1.14.201
ping  29.1.14.200

● IP&IPBR&IPRO

IPRO是IP与IPBR的绑定关系，IP绑定完IPBR后该地址就被纳入fastpath的集合

具体配置过程是 IP --> IPBR -->IPRO

1 IP配置（上面traffic部分已经完成）

2 IPBR配置

fsclish -c "add networking ipbr ipbr-id 21 ipbr-name ipbr21 committed-bandwidth 1000000 route-bandwidth 1000000 committed-sig-bandwidth 100 committed-dcn-bandwidth 100"

fsclish -c "add networking ipbr ipbr-id 22 ipbr-name ipbr22 committed-bandwidth 1000000 route-bandwidth 1000000 committed-sig-bandwidth 100 committed-dcn-bandwidth 100"

3 IPRO配置

fsclish -c "add networking ipro iface up ipbr-id 21 ip-address 29.1.14.201 owner /QNUP-0"

fsclish -c "add networking ipro iface up ipbr-id 22 ip-address 29.1.14.200 owner /QNUP-0"
