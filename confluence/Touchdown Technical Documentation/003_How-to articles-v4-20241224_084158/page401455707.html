018_How to investigate deamon execute fail

case 在执行deamon execute 总是返回第一次的值

 

  1. deamon execute keyword 引用 /rcpci/libraries/rcplib/common/connection/sctp/pysctp的_sctp.so如果FPC是64位的，引用v040x64 下面的_sctp.so 
  2. 目前/rcpci/libraries/rcplib/common/connection/sctp/pysctp/v040x64/_sctp.so 是基于libc-2.15.so编译的， 如果FPC上的FPC 是更高的版本，比如libc-2.17.so，_sctp.so需要重新编译，否则就会出现问题。 
  3. _sctp.so会引用libc.so ，并查看libc版本
       [root@24F-VFPC-001 v040x64]# /usr/bin/ldd _sctp.so
        linux-vdso.so.1 (0x00007fffef589000)
        libsctp.so.1 => /usr/lib64/libsctp.so.1 (0x00007faa00b6b000)
        libc.so.6 => /lib64/libc.so.6 (0x00007faa007c5000)
        /lib64/ld-linux-x86-64.so.2 (0x0000003bcca00000)
        [root@24F-VFPC-001 v040x64]# ls /lib64/libc.so.6 -l
        lrwxrwxrwx 1 root root 12 Oct 28 17:12 /lib64/libc.so.6 -> libc-2.15.so
      4. 这个是目前TA  Case 中用到的_sctp.so对应的原代码。
