058_How to create koji env

● create relevant files and directories


Create configure files in ecloud
touch $HOME/rcppkg.conf
touch $HOME/.gitconfig
mkdir $HOME/.ssh
mkdir $HOME/.koji.rcpdevkoji/
touch $HOME/.koji.rcpdevkoji/client.crt
touch $HOME/.waders-development-config

● make some necessary configuration

1 about rcppkg.conf

1) Download template rcppkg.conf and changing some config


$ cd ~
$ wget https://gitlabe1.ext.net.nokia.com/waders-infra/rcppkg/raw/master/etc/rpkg/rcppkg.conf -O ~/rcppkg.conf

2) get your own git_token

you don't have a token yet then create one on 
https://gitlabe1.ext.net.nokia.com/profile/personal_access_tokens


2 Create user certificate and copy client.crt to the target directory

1) create user certificate via the following link

http://rcp-dev-pl-jenkins.eecloud.dynamic.nsn-net.net:8080/job/RPM-spec-Test-Env/job/Create_user_certificate_for_developer/

2) Copy client.crt the mail you received after "User certificate"


3) create docker via waders-development.sh

Access docker
$ cd ~
$ mkdir -p sandbox/waders-development
$ cd sandbox/waders-development
$ wget https://gitlabe1.ext.net.nokia.com/waders-infra/waders-development/raw/master/waders-development.sh
$ chmod 775 waders-development.sh
$ cd ..
$ ./waders-development/waders-development.sh
Select Koji to be used
# select koji server to be used
waders-select-koji rcpdevkoji
 
# check connection to koji and configuration
koji moshimoshi

4) download the code and compiling

create your own ssh public key and upload to the following links, you can ask the developer in your team for help :

https://gerrite1.ext.net.nokia.com

https://gitlabe1.ext.net.nokia.com/

(1) Generate an ssh key


$ ssh-keygen
$ cat .ssh/id_rsa.pub


Clone subsystem and compiling
$cd ~
$rcppkg clone rcptransportmgmt
$cd rcptransportmgmt
$git checkout remotes/origin/rcp2.0 -b rcp2.0 
$rcppkg mockbuild

在koji中编译vran是注意在rcppkg.conf中修改tag或者直接加上参数：
rcppkg mockbuild  --target rcp2-container-emodp-release-wf30

If 'rcppkg clone' doesn't tale effect, you can clone the source tree manually.

Trouble shooting logs
/home/builder/rcppkgbuild/RCPTransportADK/rcppkg-results-RCPTransportADK/2.10.0/1.rcp2/root.log
/home/builder/rcppkgbuild/RCPTransportADK/rcppkg-results-RCPTransportADK/2.10.0/1.rcp2/build.log

#RCPTransportADK is an example 
#Modify /work/rcppkgbuild/RCPTransportADK/RCPTransportADK.spec
+BuildRequires:  cpputest-devel
#Modify /work/RCPTransportADK/configure.ac
+PKG_CHECK_MODULES([LIBCPPUTEST], [cpputest])
Add compile command
#RCPTransportADK is example, modify /work/rcppkgbuild/RCPTransportADK/RCPTransportADK.spec
 ./autogen.sh
 %configure
 %make_build
+%check 
+%make_check
