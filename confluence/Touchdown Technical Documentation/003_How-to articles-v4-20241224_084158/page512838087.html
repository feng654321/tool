046_How_to_install_RCP2.0

 

1. use CRT(or other tool) connect to the server you need to install RCP2.0.

 

eg: 
    server name: M2
    user: hranuser3
    password: systeM!23
After you connnect success, you curent path is as below.
[hranuser3@controller-0 ~(keystone_hranuser3)]$ pwd
/home/hranuser3

2. download the latest yam file from the SW delivery server yaml directory


 http://10.56.118.71/pilivee/RCP/rcp2.0/devel/latest/images/yaml/
Unzip  the downloaded file(rcp-heat-templates-v10.tgz) and you will see the folder as below.
Copy the files in the folder basic to the path /home/hranuser3/heat of M2 server that you connected in the step1.
Note: Curently we use the yam of basic folder.
 [hranuser3@controller-0 ~(keystone_hranuser3)]$ ll
total 564764
drwxr-xr-x 3 hranuser3 users      4096 May  2 14:40 heat

-rw-r--r-- 1 hranuser3 users 578289664 Apr  6 09:24 rcp2-ci.qcow2
drwxr-xr-x 5 hranuser3 users      4096 Apr  6 16:39 rcp-heat-templates_new

3. upload the image

 

Please fill the image location with the link that you can download the image file.

4. Generate the key file

 

nova keypair-add mykey > mykey.pem   //if this step raise error:ERROR (Conflict): Key pair 'mykey' already exists.  Means there is already has an key, you can continue the rest steps.

5. Modify the rcp-heat.env file


resource_registry:
  "RCP::Node::Management": modules/managementnode.yaml
  "RCP::Node::Application": modules/applicationnode.yaml
  "RCP::Network::Setup": modules/networksetup.yaml
  "RCP::Storage::Volume": modules/volume.yaml
  "RCP::Storage::VolumeAttachment": modules/volume_attachment.yaml
  "RCP::Storage::StorageUser": modules/storage_user.yaml
parameters:
  image: rcp2_image
  network: 545cd7e9-cf2e-40cc-8f8e-f04ccfabeff0      //ID of vnfoam
  key_name: default_key
  flavor: m1.small
  number_of_as_nodes: 2
  ntp_servers: [10.56.126.22]
  cluster_id: RCP-1234
  time-zone: Asia/Shanghai

6. create stack

 

heat stack-create -e rcp-heat.env -f rcp-heat.yaml rcp2_stack

7. get the floating IP

 

heat output-show rcp2_stack management_ip
