038_How to add SCLI command on RCP1.0

● XML

功能

●  通过xml的不同node信息，完成框架构建

● 不同的节点数据在框架体现着不同的功能点

● 可以引入不同的api接口完成具体的业务逻辑实现 

 

实例

root@MMN-0 [RCP-180]  > help cmd-tree cmd-set /networking/uplane

add networking uplane address <owner> {operator <operator> ip-address <ip-address>}

delete networking uplane address <owner> {operator <operator>}

show networking uplane address {operator <operator>}

指令关联拓扑

 

● 整个框架部署

※ xml – 掌管了scli的输入层级及方式方法，输入参数需要ldap提供时，需要通过macro引入api接口，完成读取工作，最终scli所有的输入参数也需要调用相应的api写入到ldap;

※ script – 连接ldap与xml间的数据交互，本例通过python 接口完成相应的数据操作；

※ plugin  – 主要作为ldap的client端，订阅监控ldap的相关节点的信息变动并试试获取，并通过相应接口通知到prb；

※ ldap – 主要完成ldap的配置部署工作；

※ buiid – copy.sh 完成以上各个功能组件的部署工作；

● TAC scli 解决方案

----------------------------------

1 xml 梳理，提示信息，帮助信息，相关的macro封装；

2 script python 相关API 梳理封装；

3 plugin — 根据guide 完成模板规定的api接口编写，与prb需要交互的cli接口需要完成开发；

4 ldap相关的配置工作之前team已经完成

 5 copy.sh scli相关部署文件的copy 部署需要完成
