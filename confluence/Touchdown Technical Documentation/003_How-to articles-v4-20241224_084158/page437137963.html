CCI Process

CCI Process include two stages:

cci flow and promote flow:

  1. cci flow: commit to gerrit(git push origin HEAD:refs/for/cloudVT)—>trigger CCI_{subsytstem name}_GIT (compile,ut&static analysis) ----→verify +1 ---→ review+2-----→gerrit submit-→trigger Promote flow

  2. Promote flow:gerrit submit--> trigger CCI_{subsytstem name}_TRIGGER →  trigger CCI_{subsytstem name}_FLOW(compile → make image → run FT) ----→trigger JOB PROMOTE(merge cloudVT/VirtualTrunk to cloudILTrunk/master) → CCI_FINISH(will mail Promote result to gerrit change submitter cc to change owner and send  the result to code review page)

7.how to config dependency

  • Your code still in gerrit , has not been merged into VT or cloudvt, and depend another gerrit ticket which is still not merged into VT.
    example:
    http://hzgitv01.china.nsn-net.net/#/c/75743/  depend on ticket http://hzgitv01.china.nsn-net.net/#/c/75111 (SS_ILConfmanLib)

    when cci triggered ,it will get the code from http://hzgitv01.china.nsn-net.net/#/c/75111 for SS_ILConfmanLib. 
    if you depend not only one sybsystems ,just add the numbers divide by space.

    warning: the numbers should be  belonged to different subsystems, if not , the code version will be recovered.
                    this dependency only help to checkout code from gerrit, will not  compile the subsystems you depend.

  • If you depend on testcase, just add T before the ticket number.
    example:
    http://hzgitv01.china.nsn-net.net/#/c/70270/

    when 70270 mered into VT and trigger FT, it will get the test case version from the latest commit log,.
     if FT passed,  the test  case ticket will be auto merged .

  • if you want to compile the subsystems you depend on. please configure it on ILdashboard
    example:
    http://hetran-rcp-releasenotes.eecloud.dynamic.nsn-net.net:5000/


    when you configured the depenency here, it will help to get the latest code from VT or cloudvt and make the rpms for all the subsytems in the group.
    besides,the  reversions will be record, only when all the subsystems with reversion passed FT, they will auto merged into trunk  together.

10. !!!DO NOT abort CCI jobs when FT job has been starting.

       this is a limitation of Cloud CCI , once FT starts , but aborted , there would be 2 rpm in workspace , it will cause package installation fail.

       so , please do not abort CCI jobs during FT running , if you have done , please contact I_MN_HRAN_SW3_CSDEV4_ALPHA_GMS   team to fix it. thanks!
