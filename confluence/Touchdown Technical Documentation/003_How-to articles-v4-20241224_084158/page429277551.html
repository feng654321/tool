030_how to rebuild to the newest image for ESPOO EEcloud vFPC

Step-by-step guide

Note: this guide is only for espoo eecloud vFPC

1.Check whether there is w58li.sh on /root/

[root@eec-vfpc-73 ~]# ls

ValidationToolKit_15.6

[root@eec-vfpc-73 ~]

2. If there no such file, copy from other vFPC

[root@eec-vfpc-73 ~]# scp root@10.40.112.23:/root/w58li.sh ./

The authenticity of host '10.40.112.23 (10.40.112.23)' can't be established.

ECDSA key fingerprint is 53:11:e0:15:01:a0:e9:3f:91:70:19:4e:7f:ae:f7:14.

Are you sure you want to continue connecting (yes/no)? yes

Warning: Permanently added '10.40.112.23' (ECDSA) to the list of known hosts.

root@10.40.112.23's password:            //password:nokia123

w58li.sh                                                                         100% 1575     1.5KB/s   00:00   

[root@eec-vfpc-73 ~]#

3.Source w58li.sh

[root@eec-vfpc-73 ~]# source w58li.sh

Please enter your OpenStack Password:          //password:nokia123

[root@eec-vfpc-73 ~]#

4.Check the current img version:

[root@eec-vfpc-73 ~]# mgtip=`ifconfig eth0 | grep "inet " | awk '{print $2}'`   //get the management IP

[root@eec-vfpc-73 ~]# vfpcname=`nova list | grep $mgtip | awk '{print $4}'`  //get the vFPC instance name

[root@eec-vfpc-73 ~]# nova show $vfpcname | grep image | awk '{print $4}'  //get the current vFPC img of this vFPC

HZ-SW3-centos7-vFPC-v0.6          //the vFPC73 is based on img 0.6

5.Check the newest img version:

[root@eec-vfpc-73 ~]# glance image-list
