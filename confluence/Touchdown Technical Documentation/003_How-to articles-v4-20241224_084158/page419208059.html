023_How to Install Package On Cloud

  • How to create stack?
    • 1. Click "Openstack List" to check Cloud exists or not.
    • 2. If your openstack do not exists, Click "New Openstack" and input Openstack related information then save.
    • 3. Click "FPC List" to check your FPC exists or not before add your tenant.
    • 4. Click "Tenant List" to check your tenant exists or not, If your tenant do not exist, click "New Tenant" and input tenant related information then Save.
    • 5. Click "Install Package for CRT" to install package.
    • 6. Click "Install Package", Pop out dialog input "image","yaml","deploy type" then click "Submit"
    • 7. Install task is displayed under "Task Queue"
    • 8. Results can be found under "Task Result -> SYS" click "Task ID" to see details.
    • 9. Installed Package can be found under "Stack List".
  • How to delete stack?
    • Delete specified stack
    • Delete all stacks in specified openstack
  • How to use manually created stack in CI Platform?
    • 1. Click "New Stack" button in "Stack List" page.
    • 2. Input stack information in stack edit page.

How to create stack?

1. Click "Openstack List" to check Cloud exists or not.

 

2. If your openstack do not exists, Click "New Openstack" and input Openstack related information then save.

          *Note: User Name and Password is openstack controller user/pass Not dashboard user/pass

3. Click "FPC List" to check your FPC exists or not before add your tenant.

    If your FPC do not exist, Click "New FPC" and input FPC related information then Save.

          *Notes:  1. New added FPC can be used at most in 10 Mins;

                         2. If "import error" occurred when install package, please add "PATH" environment variable in FPC configuration with value "/var/RobotFramework/rcp1.0/bin:$PATH". 

                         3. If the FPC need to run with ESPOO devices, make sure attach "espoo" label on lables field and assure the connection with gerrit server <gerrit.nsn-net.net/#/>


4. Click "Tenant List" to check your tenant exists or not, If your tenant do not exist, click "New Tenant" and input tenant related information then Save.

 

     Device Name: Special name to identify your tenant (Naming rule: M5_projectxxx)

     Tenant Name: Your tenant name on dashboard

     User Name: Your user name to login dashboard

     Password: Your password to login dashboard

     Other Target Info: Extra target ini info.

    *Note: 

 

          1. Keep "parameters:"

          2. Delete "image_name: xxx".

          3. Don't need to allocate floating IP, delete 

              oam_default_gw: 10.70.14.193

              oam_service_ip:  10.70.14.202  

              oam_service_netmask: 26

         4. The format shoud follow yaml file.

         5. If the tenant is from espoo, attach one label "espoo" in labels field, make sure both FPC and image url are from Espoo. 


5. Click "Install Package for CRT" to install package.


6. Click "Install Package", Pop out dialog input "image","yaml","deploy type" then click "Submit"

     Build Name: Support URL(e.g.: http://ip/path/image.qcow2) or Image in Openstack(e.g.: prefix_imagename)  

                           Support regular expression (e.g.: http://10.56.118.71/pilivee/RCP/rcp_common/devel/latest/images/rcp_common-ci_r.*qcow2)       

                           Support build label (e.g.: CLOUDIL_DB,   refer to how to create build label)

     Yaml: Support URL (e.g.: http://10.68.165.226/featureci/FCI2_1476429213024/build/images/yaml/rcp_large_heat.yaml)        

               Support to upload user defined yaml

               Support module yaml (e.g.: {TPLZIP}#xxx.yaml), module yaml don't support user uploaded. {TPLZIP} means the zip file, make sure the xxx.yaml file is in the zip.

 *Note:1. Image name in Openstack must match naming rule: "prefix_imagename"， yaml file name only supoort [a-zA-Z0-9_]. 

            2. Stack name will be named by image name, and it will check whether the image is installed on specified tenant.

                e.g. image name: http://10.56.118.71/CloudIL/vrnc/ci/latest/images/CLOUD_IL_RNC_16.39.0-ci_r106720-160926-145900.qcow2, stack name will be ci-deployer-CLOUD_IL_RNC_16.39.0-ci_r106720-160926-145900-IL_vRNC_Min

                So don't use common name (e.g. http://10.56.118.71/pilivee/vRNC/devel/latest/images/vrnc.qcow2) to install package, it will be  thought as same package.

            2. Don't use Hang Zhou image if you install package on Espoo device.

7. Install task is displayed under "Task Queue"

 

8. Results can be found under "Task Result -> SYS" click "Task ID" to see details.

 

9. Installed Package can be found under "Stack List".

     *Note: Make sure that if you create stack on CI Platform, please delete it on CI Platform.

                 If you delete it on openstack, it will cause problem in CRT. 

 


 

How to delete stack?

  • Delete specified stack

    Click "Stack LIst", find the specified stack name then click "delete".

 

  • Delete all stacks in specified openstack

    Click "Openstack List", find specified openstack name then click "delete stacks", it will delete all stacks on this openstack.

 

How to use manually created stack in CI Platform?

1. Click "New Stack" button in "Stack List" page.

2. Input stack information in stack edit page.

      *Note: Build name should be full image path like the install package parameter, e.g.http://halebopp.tre.noklab.net:8080/RCP/koekentta/feature/featureci_QUEUE_1482335433878/build/images/rcp_common-ci_r10705-170112-055441.qcow2 .
